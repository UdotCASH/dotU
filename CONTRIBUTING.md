# Contributing to dotU

Thank you for your interest in contributing to the dotU documentation repository!

## 🎯 Guidelines

### What We Accept
- ✅ Documentation improvements
- ✅ Usage guides and tutorials
- ✅ Integration examples
- ✅ Wallet compatibility info
- ✅ Bug reports
- ✅ Feature suggestions

### How to Contribute

1. Fork the repository
2. Create a branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Commit (`git commit -m 'Add amazing feature'`)
5. Push to branch (`git push origin feature/amazing-feature`)
6. Open a Pull Request

## 📝 Writing Guidelines

### Style
- Use clear, concise language
- Include code examples
- Add relevant links
- Test your changes

### Documentation
- Update README.md for major changes
- Add comments to code examples
- Include use cases
- Mention security considerations

## 🔧 Development

### Setup
```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/dotU.git

# Add upstream remote
cd dotU
git remote add upstream https://github.com/UdotCASH/dotU.git
```

### Syncing
```bash
git fetch upstream
git checkout main
git merge upstream/main
```

## 🐛 Reporting Issues

### Bug Reports
Include:
- Description of the issue
- Steps to reproduce
- Expected behavior
- Environment details

### Feature Requests
Include:
- Problem it solves
- Proposed solution
- Use cases
- Examples

## 📜 Code of Conduct

### Our Pledge
- Be respectful
- Be inclusive
- Be collaborative
- Be constructive
